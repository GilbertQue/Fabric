# Fabric

This is all items relating to Microsoft Fabric Workloads.

## Notebooks

- **[Get_Files_Tables_StorageSizes](Get_Files_Tables_StorageSizes.ipynb)**  
  Will run on your existing lakehouse and get the files and tables storage sizes.

- **[Adding Date Time Column to Pyspark data frame](Adding%20Date%20Time%20Column%20to%20Pyspark%20data%20frame.ipynb)**  
  Adds the current date time to an existing PySpark DataFrame.

- **[Azure Key Vault Auth with Service Principal](Azure%20Key%20Vault%20Auth%20with%20Service%20Principal.ipynb)**  
  Authenticates from an Azure Key Vault with the Service Principals needed to run the Power BI or Fabric Admin APIs.

- **[Blog - Update IR Policy](Blog%20-%20Update%20IR%20Policy.ipynb)**  
  Updates your Incremental Refresh Policy based on the dates provided in the notebook.

- **[Reading Table from Another Lakehouse](Reading%20Table%20from%20Another%20Lakehouse.ipynb)**  
  Allows you to read a Lakehouse Table from a different App Workspace.

- **[Blog-Scanner API](Blog-Scanner%20API.ipynb)**  
  Downloads the Scanner API Data to a JSON file in your Lakehouse.

- **[BLOG - Entra ID All Group Members](BLOG%20-%20Entra%20ID%20All%20Group%20Members.ipynb)**  
  Gets all the Entra Groups and their Group Members and downloads it to a JSON file in your Lakehouse.

- **[Blog - Get Files and Table Sizes](Blog%20-%20Get%20Files%20and%20Table%20Sizes.ipynb)**  
  Gets the storage size for all your files and table sizes for Lakehouses and Warehouses.

- **[Blog - Reading and Write different Lakehouses](Blog%20-%20Reading%20and%20Write%20different%20Lakehouses.ipynb)**  
  Shows you how to read from a Lakehouse A in Workspace A and write to Lakehouse B in Workspace B.

- **[Blog - Create case insensitive Warehouse](Blog%20-%20Create%20case%20insensitive%20Warehouse.ipynb)**  
  Contains the code on how to create a case insensitive Warehouse.

- **[Blog - Create warehouse with Service Principal](Blog%20-%20Create%20warehouse%20with%20Service%20Principal.ipynb)**  
  Contains the code on how to create a Warehouse using a service principal account.

- **[Blog - Get All Entra ID Groups and Users and Licenses](Blog%20-%20Get%20All%20Entra%20ID%20Groups%20and%20Users%20and%20Licenses.ipynb)**  
  Gets all the Groups, Users and User licenses in your Tenant.

- **[Blog - Get All Fabric Items - Actual Pure Python](Blog%20-%20Get%20All%20Fabric%20Items%20-%20Actual%20Pure%20Python.ipynb)**  
  Gets all the Fabric Items in your tenant using a Python-only notebook.

- **[Blog - DuckDB SQL Code to read from Delta Tables](Blog%20-%20DuckDB%20SQL%20Code%20to%20read%20from%20Delta%20Tables.ipynb)**  
  Shows you how to read data from Lakehouse tables using SQL via DuckDB.

- **[Blog - Python - DuckDB - Writing to Lakehouse Table](Blog%20-%20Python%20-%20DuckDB%20-%20Writing%20to%20Lakehouse%20Table.ipynb)**  
  Shows you how to write data from a DataFrame to a Lakehouse Table using DuckDB SQL.

- **[Blog - Python - Run DAX Query and write to LH Table](Blog%20-%20Python%20-%20Run%20DAX%20Query%20and%20write%20to%20LH%20Table.ipynb)**  
  Runs a DAX query against a semantic model and writes the DataFrame to a Lakehouse table.

- **[Blog - Python - DuckDB - Querying Multiple Tables](Blog%20-%20Python%20-%20DuckDB%20-%20Querying%20Multiple%20Tables.ipynb)**  
  Queries multiple tables using a Python notebook with DuckDB and a weather API.

- **[Blog - Python - DuckDB - Looping and write once](Blog%20-%20Python%20-%20DuckDB%20-%20Looping%20and%20write%20once.ipynb)**  
  Shows how to loop through stop dates and ramp the output once to a Lakehouse table.

- **[BLOG - Show Table with IR Policy](BLOG%20-%20Show%20Table%20with%20IR%20Policy.ipynb)**  
  Uses Semantic Link Labs to connect to the Tabular Object Model and show a table's Incremental Refresh Policy.

- **[Blog - Export all Items to OneLake](Blog%20-%20Export%20all%20Items%20to%20OneLake.ipynb)**  
  Exports all Fabric Items from your Workspace to OneLake and Azure Blob Storage.

- **[Blog - Restore from Backup Items](Blog%20-%20Restore%20from%20Backup%20Items.ipynb)**  
  Imports/Restores all Fabric Items from a folder in your Lakehouse Files section.

- **[Blog - Python Merge to Lakehouse](Blog%20-%20Python%20Merge%20to%20Lakehouse.ipynb)**  
  Pure Python notebook to merge data from API into Lakehouse table.
